NECESSARY TO CONSIDER...
----------
- the largest disadvantage of this project is the fact you would need to download all the needed files every
single time you would reload the website so that means it would be using quite a lot of wifi.
- this could be solved by loading only the data that would be needed for the player to load so it wouldn't
load every single packages - for example the maps - you don't play on all of them (you would need all the data
just in case if you're playing for longer time and you would eventually fight on all the maps)
- this could save a lot of data downloading from the server
- on the other hand this also has the advantage of no need to install the game - thus saving hard disk space
- another thing to consider is the performance loss because of the project being in a internet browser and
the communication between HTML/CSS/JavaScript and C++/C#/Python,etc.
- i tried to search for a solution and figured out this could be done by Node.js server, 
Emscriptren/WebAssembly, WebSocket, HTTP REST API and IPC(Inter-process communication = running the C++/C#
processes independently and those processes and the internet browser process - this project - would
communicate between each other).
